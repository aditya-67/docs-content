## Order APIs

The **Order** resource is used to define an ONDC-routed order placed to a single **Storefront**.

The **Order APIs** can be used toâ€“

1. Get orders for a storefront <Badge>Mandatory</Badge>
2. Update orders for a storefront <Badge>Mandatory</Badge>

### Get orders for a storefront

**Sample Request**

<table>
  <tr>
    <th>Method</th>
    <td>
      <code>GET</code>
    </td>
  </tr>
  <tr>
    <th>URL</th>
    <td>
      <code>/api/storefronts/{"storefront_id"}/orders</code>
    </td>
  </tr>
</table>

**Sample Response**

<CodeBlockWithCopy>{`
{
    [
        {
            "order_id": "string", 
            "order_state": "string",
        }, 
        {
            "order_id": "string", 
            "order_state": "string",
        }
    ]
}
`}</CodeBlockWithCopy>
